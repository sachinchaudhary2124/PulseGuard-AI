# 🚨 PulseGuard AI

## AI-Powered API Failure Detection & Debugging System

PulseGuard AI is an autonomous AI observability platform designed to monitor API health, detect anomalies in real-time, analyze infrastructure incidents, and generate intelligent root-cause insights for engineering teams.

Built during a 2-day hackathon, this system simulates a real-world AI-powered observability dashboard used by DevOps, SRE, and backend engineering teams.

# 🎯 Problem Statement

Modern engineering teams struggle to identify:

- API failures
- Server crashes
- High latency
- Gateway timeouts
- Backend instability

before customers report issues.

Traditional monitoring systems generate logs and alerts, but engineers still spend valuable time manually debugging incidents.

PulseGuard AI solves this problem by combining:
- Real-time anomaly detection
- AI-powered root cause analysis
- Executive incident summarization
- Infrastructure observability
- Intelligent debugging insights

into a single autonomous monitoring platform.

# 🚀 Key Features

## ✅ Real-Time API Monitoring
Continuously simulates live API traffic and monitors endpoint performance.

## ✅ AI-Powered Incident Analysis
Uses OpenAI GPT models to analyze anomalies and generate intelligent debugging insights.

## ✅ Executive Incident Summary
Generates business-level summaries understandable by leadership and engineering managers.

## ✅ Smart Anomaly Detection
Detects:
- High latency
- HTTP 5xx server failures
- Critical infrastructure issues
- Performance degradation

## ✅ Interactive Dashboard
Built using Gradio with:
- Live charts
- Endpoint filters
- Dynamic monitoring panels
- Severity tracking

## ✅ Realistic Observability Simulation
Simulates production-grade API incidents across:
- /login
- /payment
- /orders
- /inventory
- /search
- /profile
